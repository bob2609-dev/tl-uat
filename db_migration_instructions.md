# Database Configuration Standardization Instructions

## Overview

As part of the security remediation effort for the TestLink application, we need to standardize all database connection approaches to use a single configuration file. This addresses the "Exposed Configuration Files" vulnerability identified in the VAPT report.

## Current Issues

The codebase currently uses multiple database configuration approaches:

1. **Main Configuration**: `config_db.inc.php` with standard TestLink constants (DB_HOST, DB_USER, etc.)
2. **Root-level Alternate Configuration**: `db_config.php` with variables ($db_host, $db_user, etc.) using different credentials
3. **Execute Directory Configuration**: `lib/execute/db_config.php` with variables ($db_host, $db_user, etc.)

## Migration Steps

### 1. Modified Files

The following files need to be modified to use `config_db.inc.php` instead of `db_config.php`:

#### a. `bugs_view.php`

Find:
```php
// Load database configuration
require_once __DIR__ . '/db_config.php';
// Define constants for SimpleDBHandler
define('DB_TYPE', 'mysql');
define('DB_HOST', $db_host);
define('DB_USER', $db_user);
define('DB_PASS', $db_pass);
define('DB_NAME', $db_name);
define('DB_TABLE_PREFIX', '');
```

Replace with:
```php
// Load database configuration
require_once(__DIR__ . '/config_db.inc.php');
// DB_TYPE, DB_HOST, DB_USER, DB_PASS, DB_NAME and DB_TABLE_PREFIX are now available from config_db.inc.php
```

#### c. `lib/execute/get_testcase_data_simple.php`

Change 1:
Find:
```php

```

Replace with:
```php

```

Change 2:
Find:
```php

    // Create database connection
    $db = new mysqli($db_host, $db_user, $db_pass, $db_name);
```

Replace with:
```php

    // Create database connection
    $db = new mysqli(DB_HOST, DB_USER, DB_PASS, DB_NAME);
```

## Implementation Details

### Step-by-Step Migration Process

1. **Make a backup** of the entire TestLink directory before making any changes

2. **Verify config_db.inc.php** contains the correct database connection details:
   ```php
   <?php
   // Include PHP 8 compatibility layer
   require_once('custom/inc/php8_init.php');
   // Automatically Generated by TestLink Installer
   define('DB_TYPE', 'mysql');
   define('DB_USER', 'tl_uat');
   define('DB_PASS', 'tl_uat269');
   define('DB_HOST', 'localhost');
   define('DB_NAME', 'tl_uat');
   define('DB_TABLE_PREFIX', '');
   ```

3. **For each file that uses db_config.php**, make the following changes:

   * **Modify the require statement**:
     * Change from: `require_once(__DIR__ . '/db_config.php');` 
     * Change to: `require_once(__DIR__ . '/[relative_path_to]/config_db.inc.php');`
     * Be sure to adjust the relative path accordingly for each file

   * **Replace variable references**:
     * Change from: `$db_host`, `$db_user`, `$db_pass`, `$db_name`
     * Change to: `DB_HOST`, `DB_USER`, `DB_PASS`, `DB_NAME`

   * **Update any logging code**:
     * Change from: `file_put_contents($log_file, "Attempting database connection to {$db_host}...\n", FILE_APPEND);`
     * Change to: `file_put_contents($log_file, "Attempting database connection to " . DB_HOST . "...\n", FILE_APPEND);`

4. **Test each file** after making changes to ensure it works correctly

5. **Remove the db_config.php files** only after confirming all changes work correctly

## Future Database Configuration Changes

To modify database connection details in the future:

1. **ONLY edit the `config_db.inc.php` file** - this is the single source of truth for database connections
2. The file is located at the root of the application
3. Modify the following constants as needed:
   - `DB_TYPE`: Database type (mysql)
   - `DB_HOST`: Database host (localhost)
   - `DB_USER`: Database username
   - `DB_PASS`: Database password
   - `DB_NAME`: Database name
   - `DB_TABLE_PREFIX`: Table prefix (if any)

## Security Best Practices

1. After completing the migration, verify that no database credentials are hardcoded in any other files
2. Consider implementing a secure vault or environment-based configuration for production environments
3. Remove any debug logging of database credentials in production code
4. Ensure file permissions are restricted appropriately on configuration files
